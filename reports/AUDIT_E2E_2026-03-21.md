### 4.3 Calitatea Codului

- **Arhitectura client**: Next.js 14 App Router cu separare clară între pagini publice și dashboard autentificat. Zustand pentru state management (simplu, fără over-engineering).
- **ApiClient centralizat**: `lib/api.ts` conține toate cele 53 de endpoint-uri ca metode typed pe o singură clasă `ApiClient`. Pattern excelent pentru mentenabilitate.
- **Error handling robust**: Response interceptor în `api.ts` tratează toate codurile HTTP (401, 403, 404, 422, 429, 500, 503) cu mesaje în română. La 401 pe pagini protejate, face redirect automat la `/login`.
- **HTTPS enforcement**: `api.ts` forțează HTTPS în producție prin `getSecureApiUrl()`.
- **Componente onboarding**: 6 componente separate pentru pașii wizard — cod curat și testat.
- **Cod potențial duplicat**: `CATEGORY_LABELS` definit inline în `dashboard/page.tsx` și posibil redefinit în alte componente — ar trebui mutat în `lib/categories.ts`.
- **TODO-uri**: Nu s-au detectat TODO-uri critice nerezolvate.
- **Calitate generală**: cod TypeScript tipizat, fără `any` excesiv. Utilizare corectă a hooks-urilor React.

### Top 5 Probleme Critice

1. **[MAJOR] `phpinfo-test.php` pe server** — Fișierul `backend/wp-content/plugins/teinformez-core/phpinfo-test.php` expune informații detaliate despre configurația PHP (versiuni, module, căi). Dacă este accesibil public pe VPS2, este o vulnerabilitate serioasă. Trebuie șters imediat atât din cod cât și de pe server.

2. **[MEDIU] JWT în js-cookie (accesibil JavaScript)** — Tokenul `teinformez_token` poate fi accesat de orice JavaScript care rulează pe pagină (risc XSS). Pentru o platformă cu date personale (preferințe, abonamente), ar trebui migrat la httpOnly cookie cu endpoint de refresh.

3. **[MEDIU] Security Alert nerezolvat complet** — `SECURITY_ALERT.md` are un checklist cu mai multe iteme nemarcate ca done (revoke key vechi, update Vercel env vars etc.). Trebuie verificate toate itemele din checklist.

4. **[MINOR] `CATEGORY_LABELS` duplicat** — Dicționarul de etichete pentru categorii este definit inline în `dashboard/page.tsx` și posibil în alte locuri. Ar trebui centralizat în `lib/categories.ts` (care deja există).

5. **[MINOR] URL-ul de producție neconfirmat în audit** — README menționează `teinformez.eu` ca URL de producție pe VPS2, dar nu există un `.env` cu `NEXT_PUBLIC_WP_API_URL` configurat detectabil. Trebuie verificat că variabilele de mediu sunt corect configurate atât pe Vercel cât și pe VPS2.

### Recomandări Next Steps

1. Sterge `phpinfo-test.php` de pe server și din repository.
2. Verifică și bifează toate itemele din `SECURITY_ALERT.md`.
3. Migrează token-ul JWT la httpOnly cookie prin modificarea endpoint-ului de login și adăugarea unui endpoint `/auth/cookie`.
4. Centralizează `CATEGORY_LABELS` în `lib/categories.ts`.
5. Adaugă un test E2E real (Playwright sau Cypress) care testează flow-ul complet: register → onboarding → dashboard → citire articol → abonament email.
